PHP Usage

PHP GnuPG Extension

可以參考 GnuPG 官方extension, 安裝方法在網路上可以找到很多文章, 在這邊就簡單說明一下, 但最後還是沒有使用PHP GnuPF Extesion, 而是直接在PHP code下 shell_exec, 主要遇到3個問題, 研究了很久:

  1. 安裝超級麻煩, 雖然安裝好也成功啟動了
  2. 解密時需要私鑰(private key), 如果在一開始安裝時有設定密碼, 那gpg2會詢問妳密碼, 目前PHP gnuPG 不支援 gpg2 密碼輸入, 可以參考下面有人討論, 所以最好的方法就是不鑰設定私鑰(private key)密碼
  1. 更改壓縮資料方式, gnupg在加密前會先壓縮資料, default 使用zlib壓縮, 但當遇到要求鑰zib壓縮時就無法靠gnuPG extension取修改壓縮方式(或是我沒找到, 但浪費了我一個工作天)

綜合以上三個問題, 我放棄使用PHP GnuPG Extension

其他問題

Logout OR Invalidate Token

來源: Invalidating JSON Web Tokens
為了要解決 登出 或者 刪除token 問題, stackover flow 討論以下幾種可能的解法,

  1. 直接在 client 將此 token 從 local storage 或者 cookies 移除
    這個方法對server端的安全沒有幫助, 攻擊者可以在第一次登入取得token時就先將token給儲存下來, server端還是會讓這組token有效, 對比session destroy, server端會讓session完全無效, 但這個方法對一般使用者來說確實是可以實作 登出功能

  2. 建立一個token黑名單
    可以把想要刪除或者使之無效的 token (或 jti claim) 加入黑名單, 直到 token 無效為止 (在 Redis 的 key-value storage 有很好的實作), 這樣在每次操作都需要比對 token 是否在 DB (Redis) 的黑名單, 這樣有點降低使用 token-bases-auth的必要, 因為每次檢查 token 都還要對 DB 做操作, 不過好處是儲存 token 的空間 可能 會更小, 因為DB只要儲存還有效的 token, 當 token 無效時 DB 會自動 expire delete 掉

  3. 讓token有效時間很短並且時常更換
    簡單說可以設定一個只有一個有效期間很短的token, 並且在每次操作時都更換有效時間產生新的token, 這樣token就會保持最新狀態, 但這個方法的問題在於沒辦法保持很長久的用戶登入狀態 ( 要取決於盜期間格的時間 )

緊急狀況

當緊急狀況發生時, 想要讓所有登入無效, 最快的辦法是直接改變jwt secret, 這樣所有token都會變成 Invalidate token

Instagram Graph API

帳戶種類

  1. Instagram Business Accounts
  2. Instagram Creator Accounts

帳戶功能

  1. 獲取媒體 (get media)
  2. 發布媒體 (publish media)
  3. 管理回覆留言評論
  4. 識別被 @mention 到的用戶
  5. 找出帶有 #(hashtag) 標記到的媒體
  6. get basic metadata about other Instagram Businesses and Creators (metada包含哪些?)
  7. get metrics about other Instagram Businesses and Creators (metrics包含哪些?)

通常功能

  1. 獲取和管理已發布的照片，視頻和故事
  2. 獲取有關其他Instagram Business用戶和創作者的基本數據
  3. 審核評論及其回复
  4. 測量媒體和個人資料的互動
  5. 發現雜亂無章的媒體
  6. 發現@提及
  7. 發布照片和影片（目前處於封閉Beta版）

建議

如果只是要取得基本個人資料或照片等可以只使用instagram Basic Display API

限制

  1. 該API無法訪問Intagram消費者帳戶（即非企業或非創作者Instagram帳戶）。 如果您要為消費者用戶構建應用程序，請改用Instagram基本顯示API。
  2. 內容發佈僅適用於Instagram業務用戶。
